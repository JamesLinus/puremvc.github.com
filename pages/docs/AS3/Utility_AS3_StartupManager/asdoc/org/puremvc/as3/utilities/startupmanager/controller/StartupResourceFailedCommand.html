Provides a command that is invoked when an application fails to load a startup resource. The failure occurs within the application's own resource proxy when it is carrying out the load activity. The command invocation is via a Notification sent by the application. The body of the Notification MUST MUST MUST identify the resource, using the application's resource proxy name. This name can be simply a String object or can be embedded in a FailureInfo object.

This command must be registered for all the relevant notifications, for example in the application's concrete facade.
